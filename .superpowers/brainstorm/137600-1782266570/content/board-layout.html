Board Layout: How Columns + Cards Render Per Breakpoint

Mobile (<768px) / Tablet (768–1199px) / Desktop (≥1200px)

Desktop (≥1200px) — Settled

Horizontal scrollable columns. Cards stacked vertically per column. Drag between columns. This is the standard Kanban board. No debate here.

Tablet (768–1199px) — Two Options

A

Compact Horizontal Scroll

Same as desktop but narrower columns. 2-3 columns visible at a time. Horizontal swipe to see more. Cards are compact — title + badge only.

B

Column Picker + Single Column

Show one column at a time. Top bar = column tabs (Backlog | Spec-ing | Planned | ...). Swipe or tap to switch columns. Cards full-width.

Mobile (<768px) — Three Options

A

Card List (Grouped by Column)

Single scrollable list. Column headers as section dividers. Cards = full-width rows with title + type badge + assignee avatar. Tap card → modal.

Simple. Familiar. Like a grouped email inbox. Loses the spatial Kanban feel.

B

Column Swipe (Tinder-Style)

One column at a time. Swipe left/right to change columns. Cards stacked vertically. Dot indicator shows column position.

Feels native. Good for few columns. Annoying with 6+ columns.

C

Mini Kanban (Horizontal Scroll)

Same as desktop but tiny. Narrow columns (card = title only, 2 lines max). Horizontal swipe. Pinch to zoom column if needed.

Preserves Kanban mental model. Cramped on small screens. Hard to read.

Recommendation

Tablet: A (compact horizontal) — keeps Kanban feel, 2-3 columns visible is enough context.

Mobile: A (card list grouped by column) — most readable, works with any column count, familiar UX pattern. The Kanban spatial model is a desktop affordance; forcing it on mobile hurts usability.
