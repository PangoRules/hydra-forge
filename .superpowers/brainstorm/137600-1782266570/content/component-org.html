Component Organization: How to Structure the Tree

Phase 3 has 15+ components. Flat folder or feature-grouped?

A

Feature-Grouped (Recommended)

components/
  auth/
    LoginForm.vue
    SetupForm.vue
  project/
    ProjectList.vue
    ProjectCreateModal.vue
    ProjectSidebar.vue
  board/
    BoardView.vue
    BoardColumn.vue
    BoardCard.vue
    BoardMobileList.vue
    ColumnHeader.vue
  card/
    CardModal.vue
    CardDescription.vue
    CardChecklist.vue
    CardComments.vue
    CardAttachments.vue
    CardDependencies.vue
    CardMetadata.vue
  spec/
    SpecEditor.vue
    SpecVersionHistory.vue
  plan/
    PlanEditor.vue
    PlanVersionHistory.vue
  shared/
    UserAvatar.vue
    MarkdownEditor.vue
    ConfirmDialog.vue
    ErrorToast.vue
    KeyboardShortcutOverlay.vue

Clear ownership. Easy to find. Scales to Phase 4+ features.

B

Flat + Prefix

components/
  AuthLoginForm.vue
  AuthSetupForm.vue
  BoardView.vue
  BoardColumn.vue
  BoardCard.vue
  CardModal.vue
  CardChecklist.vue
  ...

Simpler. No nested folders. Gets messy past 20 components.

C

Atomic Design

components/
  atoms/   Button, Badge, Avatar
  molecules/ CardRow, ColumnHeader
  organisms/ BoardView, CardModal
  templates/ ProjectLayout
  pages/    (in pages/ dir)

Overkill for Phase 3. Good for design systems. Adds friction.

Composables (app/composables/)

useApi.ts          — openapi-fetch client + middleware
useAuth.ts         — login/logout/isAuthenticated
useAuthToken.ts    — cookie token storage
useBoard.ts        — Pinia board store (columns, cards, mutations)
usePresence.ts     — SignalR PresenceHub connection
useRealtime.ts     — SignalR BoardHub connection + event dispatch
useKeyboard.ts     — keyboard shortcut registry
useErrorToast.ts   — ProblemDetails → toast

Pages (app/pages/)

pages/
  index.vue         — redirect to /projects or /login
  login.vue         — login form
  setup.vue         — first-run admin setup
  projects.vue      — project list
  projects/
    [id]/
      board.vue     — board view (main workspace)

Recommendation: A

Feature-grouped. Scales naturally. Each folder = one domain concept. Matches the backend's Clean Architecture separation (Projects, Cards, Specs, Plans).
