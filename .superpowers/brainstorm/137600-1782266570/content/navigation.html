Screen Architecture: How Should Navigation Work?

Mobile-first. Three breakpoints: <768px / 768–1199px / ≥1200px. Pick the navigation model.

What screens exist in Phase 3?

Auth

Login, first-run admin setup

Projects

List view, create project

Board

Columns + cards, drag-and-drop

Card Detail

Modal or page? Title, desc, checklist, comments, specs, plans, deps

A

Sidebar + Content (Traditional SPA)

Persistent sidebar (collapsible on mobile). Board = main content area. Card detail = slide-over panel or modal. Projects list = separate route.

Mobile: bottom nav bar replaces sidebar. Card detail = full-screen page.

B

Full-Screen Pages (Mobile-Native)

Every view = full page. Board → tap card → navigate to card page. Back button returns to board. No modals, no slide-overs.

Desktop: card detail renders as side panel (same component, different layout slot).

C

Hybrid: Board-Centric + Modal Detail

Board is the hub. Card detail = modal on all sizes. Specs/plans open in modal too. Projects list = separate page. Minimal route changes.

Mobile: modal goes full-screen. Desktop: centered modal with backdrop.

Key trade-off

Modal vs Page for card detail: Modal keeps board context visible (desktop), but complex on mobile (scroll, keyboard). Page gives clean mobile UX but loses board context on desktop. Nuxt UI's USlideover could bridge this — slide-over on desktop, full page on mobile.
