Card Detail Modal: Layout & Sections

Card has 10+ sections. How to organize without overwhelming?

What's in a card?

Title Description (markdown) Type (Task/Bug/Epic) Column Assignees Checklist Comments Attachments Specs Plans Dependencies Parent Epic Due date
A

Two-Column Split (Desktop) / Stacked Tabs (Mobile)

Desktop: Left 60% = description + comments. Right 40% = metadata sidebar (type, assignees, checklist, specs, plans, deps).

Mobile: Tabs at top: "Details" | "Checklist" | "Comments" | "Related". Each tab = vertical scroll.

Linear-style. Most common pattern. Metadata always visible on desktop.

B

Single Scroll + Collapsible Sections

Everything in one vertical scroll. Sections are collapsible accordions (Description, Checklist, Comments, Attachments, Specs, Plans, Dependencies).

Same layout on all sizes — just narrower on mobile. No tabs, no sidebars.

Simplest to build. Can get long. Collapse state = user preference.

C

Priority-Ordered Stack

Top: title + type + column (always visible). Then: description (expanded). Then: checklist (if items exist). Then: comments (always). Below fold: specs, plans, attachments, deps — each as expandable section.

Mobile: same order, just full-width. "Quick actions" FAB for move/assign/archive.

Content-first. Metadata pushed down. Good for reading. Harder to edit metadata.

Recommendation: A

Two-column on desktop gives persistent metadata visibility — you can edit type/assignees while reading description. Tabs on mobile prevent endless scroll. This is what Linear, Jira, and Notion all converged on.
